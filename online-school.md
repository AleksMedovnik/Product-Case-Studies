## Краткое описание проекта

Образовательная платформа с AI-ассистентом для изучения языков.

### Ближайшие аналоги/референсы

- Lingualeo
- Duolingo

## Платформы и стек

- **Веб-приложение** (запись на курс и видеоконференции)
  - Frontend: React
  - Backend: ASP.NET, Node.js, Mediasoup (видеостриминг, ядро на Rust)

- **Админ-панель**
  - Frontend: Vue.js
  - Backend: ASP.NET

- **Мобильное приложение** (уроки)
  - Frontend: Flutter
  - Backend: ASP.NET

- **AI-модели**: LLM, ASR, TTS
- **Развертывание**: AWS (в 2024 г. рассматривался Selectel, но инфраструктура была недостаточна)
- **База данных**: Postgres, Redis
- **Контейнеризация**: Docker

## AI-решения

### Бизнес-проблемы

1. Пользователям требовалась помощь при выполнении заданий; также хотели добавить свободное общение с чат-ботом.
2. Алгоритмическая проверка не подходила для некоторых типов заданий (например, сочинений). Требовалась автоматизированная проверка грамматики и лексики.
3. Создание новых курсов и упражнений требовало привлечения методистов (английский, арабский языки).
4. Для упражнений на аудирование использовались сторонние решения или голоса преподавателей.

### Бизнес-решения

1. Чат-бот в мобильном приложении.
2. Нейросеть для проверки заданий, требующих участия преподавателя.
3. Нейросеть для генерации новых упражнений.
4. Нейросеть для озвучивания текста в упражнениях на аудирование.

### Почему AI

- Участие человека нерентабельно в долгосрочной перспективе (24/7, высокая стоимость).
- Нейросеть повышает лояльность пользователей (мгновенный ответ, общение в любое время).
- Генерация упражнений оптимизирует работу методистов.
- Собственный TTS даёт независимость от сторонних решений и гибкость в выборе голосов.

---

## Стратегия выбора решения

Выбраны **pre-trained модели**, развёрнутые в собственной среде, а не подписка на ChatGPT-4.

**Факторы**:
- Контроль поведения и воспроизводимость
- Европейская логика объяснения арабского языка у ChatGPT-4
- Риск утечки данных
- Непредсказуемые расходы
- Зависимость от множества сторонних решений
- Инвестор соглашался финансировать только кастомное решение на базе open-source

---

## Обоснование выбора модели

### LLM

**Тестировалось**: 38 моделей

**Ключевые метрики**:
- Параметры (млрд)
- Размер контекста (токен)
- Скорость ответа (токен/сек)
- Стоимость ресурсов ($)
- Объём на диске (GB)
- Загрузка GPU (vRAM)
- Перплексия
- Качество ответов (оценка методистов)
- Лицензия

**Выбраны**:
- LLM En: `meta-llama/Meta-Llama-3-8B-Instruct`
- LLM Ar: `FreedomIntelligence/AceGPT-v1.5-13B-Chat`

---

### ASR

**Тестировалось**: 20 моделей

**Ключевые метрики**:
- Скорость (слово/сек, токен/сек)
- RTF (Real-Time Factor)
- Загрузка GPU (vRAM)
- Average WER
- Average WER по EEE (акцентная вариативность)
- Лицензия

**Дополнительно (ретроспектива)**:
- CER (для арабского)
- Robustness tests (шум, эхо, микрофон)

**Выбраны**:
- ASR En: `distil-whisper/distil-large-v3`
- ASR Ar: `openai/whisper-large-v3`

---

### TTS

**Тестировалось**: 9 моделей

**Ключевые метрики**:
- Скорость (токен/сек)
- Поддержка мужского/женского голоса
- Возможность клонирования голоса
- Загрузка GPU (vRAM)
- MOS (оценка методистов)
- Лицензия

**Дополнительно (ретроспектива)**:
- Latency
- RTF
- Prosody / Naturalness
- Multispeaker consistency
- Cold start time

**Выбраны**:
- TTS En: `myshell-ai/MeloTTS-English` + OpenVoice
- TTS Ar: `StyleTTS2`

---

## Подготовка и тренировка моделей

### LLM En (`meta-llama/Meta-Llama-3-8B-Instruct`)

- **Методы**: Prompt-engineering
- **Проблемы**: Галлюцинации при проверке заданий
  - **Решения**: Жёсткие инструкции, JSON-ответы, снижение температуры
  - **Итог**: Полностью победить не удалось, но улучшили

### LLM Ar (`FreedomIntelligence/AceGPT-v1.5-13B-Chat`)

- **Методы**: Квантизация (4bit), Prompt-engineering
- **Проблемы**: Путаница в грамматике, галлюцинации, долгий ответ
  - **Решения**: RAG, ограничение длины ответа, снижение температуры, оптимизация attention
  - **Итог**: Улучшили, но ошибки в арабской грамматике остались; перепроектирование бэкенда под стриминг

### ASR En (`distil-whisper/distil-large-v3`)

- Работала из коробки, проблем не было

### ASR Ar (`openai/whisper-large-v3`)

- **Проблемы**: Плохое распознавание диакритики
  - **Решение**: Fine-tune на датасете с диакритизированными транскрипциями
  - **Итог**: Значительное улучшение

### TTS En (`myshell-ai/MeloTTS-English`)

- **Проблемы**: Роботизированное произношение
  - **Решение**: Добавлен OpenVoice (style-слой)
  - **Итог**: Речь стала эмоциональной и естественной

### TTS Ar (`StyleTTS2`)

- **Проблемы**: Неправильная диакритика, шумы, шипение
  - **Решения**: Предварительная диакритизация через LLM, ограничение домена, dynamic range compression
  - **Итог**: Проблемы устранены в пределах нормы

## Архитектура бизнес-решений

### 1. Чат-бот (мобильное приложение)

**Текстовое сообщение**:
1. Frontend → запрос в backend
2. Backend формирует prompt + контекст
3. Backend → LLM
4. LLM → ответ
5. Backend сохраняет в БД
6. Backend → ответ в мобильное приложение
7. UI отображает текст
8. TTS (асинхронно) → голосовое сообщение

**Голосовое сообщение**:
1. Аудио → ASR
2. Далее по схеме текстового сообщения

---

### 2. Проверка заданий (LLM)

1. Frontend → текст задания в backend
2. Backend формирует prompt (критерии, структура)
3. Backend → LLM
4. LLM → структурированный ответ (фидбек + балл)
5. Backend сохраняет в БД
6. Backend → ответ в мобильное приложение
7. UI отображает фидбек и оценку

---

### 3. Генерация упражнений (админ-панель)

1. Админ → параметры (язык, уровень, тема и т.д.)
2. Backend формирует prompt
3. Backend создаёт задачу в очереди, сохраняет статус
4. Backend → UI: "Generation started"
5. Worker → LLM
6. LLM → структурированный контент
7. Сохранение в БД
8. UI → готовое задание (доступно для редактирования)

---

### 4. Озвучивание текста (TTS)

1. Админ создаёт упражнение с текстом
2. Backend готовит конфиги (язык, голос, скорость и т.д.)
3. Задача в очереди → статус в БД
4. Worker → TTS
5. TTS → аудио
6. Post-processing
7. Сохранение в MinIO + ссылка в БД
8. UI → аудио для проверки
9. Возможность переозвучить при необходимости

## Оптимизация ресурсов

### Инстансы

Выбрано **4 отдельных GPU-инстанса** для изоляции нагрузки, масштабируемости и надёжности.

| Инстанс | Модели | Тип | Ориентировочная стоимость |
|---------|--------|-----|----------------------------|
| 1 | LLM-En | `g4dn.xlarge` | ~$615/мес |
| 2 | LLM-Ar + RAG | `g4dn.xlarge` | ~$615/мес |
| 3 | TTS-En + TTS-Ar + ASR-En + ASR-Ar | `g4dn.xlarge` | ~$615/мес |
| 4 | Speaking-arabic (ASR + LLM + TTS) | `g6.xlarge` | ~$615/мес |

**Итого**: ~$2460/мес (при 24/7 работе)

---

### Проблемы и ошибки

- Слишком оптимистичный расчёт стоимости
- Конфликты моделей на одном GPU → перешли к 4 инстансам
- Неочевидные расходы AWS (load balancer, MinIO, data transfer)
- Инстансы простаивают, но оплата идёт 24/7
- GPU сложнее масштабировать, чем CPU
- После квантизации качество иногда падало
- MLOps изначально выполняли ML-инженеры → не хватало автоматизации
- Отсутствие мониторинга (токены/сек, latency, GPU utilization, VRAM)
- Отсутствие централизованного логирования и автоматических тестов качества

---

### Обоснование стоимости

- Инстансы работают 24/7 (холодный старт медленный)
- Планируется горизонтальное масштабирование: основной инстанс + временный при пиках
- Инфраструктура рассчитана на 20–40 одновременно активных пользователей и 100–300 DAU

## Экономический эффект

> Точные цифры не приводятся, так как AI — core-функция с самого начала.

### Чат-бот
- Повышение Retention и LTV
- Снижение Churn

### Проверка заданий
- Повышение креативности заданий
- Повышение вовлечённости

### Генерация упражнений
- Сокращение времени на создание урока:
  - **Было**: ~3 часа (методист)
  - **Стало**: ~25 минут (корректировка)

### Озвучивание текста
- Отказ от платных дикторов и сторонних решений
- Собственный TTS, инстансы уже оплачены

---

## Выводы

- Pre-trained модели — оптимальный выбор для контроля, безопасности и кастомизации.
- Основные сложности: арабский язык, латентность, стоимость GPU, галлюцинации.
- Архитектура с отдельными инстансами оправдана, но требует постоянного мониторинга и оптимизации.
- AI-функции значительно ускорили создание контента и повысили качество обучения.
